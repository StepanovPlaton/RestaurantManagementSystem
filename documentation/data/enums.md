# Перечисления и справочники

Справочные значения, зафиксированные в миграциях Flyway и коде backend.

## Тип пользователя (`users.user_type`)

| Значение   | Описание |
|------------|----------|
| `CLIENT`   | Клиент ресторана (мобильное приложение) |
| `EMPLOYEE` | Сотрудник ресторана (админ, менеджер или курьер) |

Задаётся при создании записи в `users` и определяет, какой JWT-контур используется (`jwtClient` / `jwtEmployee`).

## Роли сотрудников (`roles.name`)

Справочник заполняется миграцией `V3__seed_roles.sql`:

| `id` | `name`    | Описание |
|------|-----------|----------|
| 1    | `ADMIN`   | Полный доступ: меню, блюда, ингредиенты, сотрудники, роли, статусы заказов |
| 2    | `MANAGER` | Заказы, клиенты, назначение курьеров, управление меню и блюдами |
| 3    | `COURIER` | Свои заказы, смена статуса доставки, рабочий статус |

Используется в `GET /employees?role_id=2` и при создании сотрудника (`role_id` в теле запроса).

Роль повара (`COOK`) в системе **не используется**.

Связь: `employees.role_id` → `roles.id`. В JWT для сотрудника authority совпадает с `roles.name`.

## Статусы заказа (`order_statuses.status_name`)

Справочник заполняется миграцией `V4__seed_order_statuses.sql`:

| Значение      | Типичный смысл |
|---------------|----------------|
| `NEW`         | Заказ создан, ожидает обработки |
| `COOKING`     | Заказ готовится на кухне |
| `DELIVERING`  | Заказ передан курьеру / в доставке |
| `DELIVERED`   | Заказ доставлен |
| `CANCELLED`   | Заказ отменён |

Текущий статус заказа определяется последней записью в `order_status_history` (по `changed_at`).

## Именование JSON в API

Backend сериализует поля в **snake_case** (`first_name`, `status_name`, `total_price`) — см. `spring.jackson.property-naming-strategy=SNAKE_CASE` в `application.properties`.
