# Роль: Курьер (COURIER)

Доставка назначенных заказов. Mobile web с нижней навигацией «Заказы» | «Профиль».

## Возможности

- Просмотр списка своих заказов
- Детальный просмотр заказа и смена статуса доставки
- Переключение рабочего статуса (`is_working`) — доступен для назначения новых заказов
- Редактирование своего профиля

## Страницы

### Вход

См. [index.md](index.md#общие-экраны). API: `POST /auth/employees/login`.

### Мои заказы

- **Назначение:** список заказов, назначенных курьеру.
- **Элементы:** переключатель «Статус: работаю»; карточки заказов (номер, адрес, число блюд, сумма, метка статуса).
- **Действия:** открыть карточку заказа; переключить рабочий статус.
- **API:** `GET /orders` (фильтрация по `courier_id` на backend), `PATCH /employees/{id}` для `is_working`
- **Макет:** [fig-23](../пояснительная-записка/assets/fig-23-prototip-stranitsy-spiska-zakazov-kurera.jpg)

### Карточка заказа

- **Назначение:** детали заказа и переход к следующему статусу.
- **Элементы:** адрес, клиент, состав, шкала статусов («Принят» → «В пути» → «Доставлен»), кнопка смены статуса.
- **API:** `GET /orders/{id}`, `POST /orders/{orderId}/status-history`
- **Макет:** [fig-24](../пояснительная-записка/assets/fig-24-prototip-stranitsy-kartochki-zakaza-kurera.jpg)

### Подтверждение смены статуса

- **Назначение:** подтверждение доставки перед записью в историю.
- **API:** `POST /orders/{orderId}/status-history` (статус `DELIVERED` или `DELIVERING`)
- **Макет:** [fig-25](../пояснительная-записка/assets/fig-25-prototip-ekrana-podtverzhdeniya-smeny-statusa-zaka.jpg)

### Профиль

- **Назначение:** данные курьера, дублирование переключателя смены, выход.
- **Элементы:** аватар, ФИО, телефон; «На смене сегодня»; «Личные данные»; **«Справка»** (`/courier/help`), **«О разработчиках»** (`/courier/about`); «Выйти».
- **API:** `GET/PATCH /employees/{id}` (свой профиль)
- **Макет:** [fig-26](../пояснительная-записка/assets/fig-26-prototip-stranitsy-profilya-kurera.jpg)

## Навигация

Нижняя панель: «Мои заказы» ↔ «Профиль»; из списка → карточка → подтверждение → список — [fig-33](../пояснительная-записка/assets/fig-33-navigatsionnaya-model-interfeysa-kurera.jpg).

## Статусы в UI vs API

На макетах курьера упрощённая шкала (Принят / В пути / Доставлен). В API используются коды `NEW`, `COOKING`, `DELIVERING`, `DELIVERED`, `CANCELLED` — см. [enums.md](../data/enums.md). Справочник `GET /order-statuses` доступен только **Admin**; курьер меняет статус через `POST /orders/{orderId}/status-history`.
