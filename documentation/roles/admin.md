# Роль: Администратор (ADMIN)

Полный доступ к управлению рестораном: меню, блюда, ингредиенты, персонал, справочники. Desktop web-приложение с боковым меню.

## Возможности

- CRUD меню и привязка блюд к меню
- CRUD блюд, фото блюд, состав (ингредиенты)
- CRUD ингредиентов
- CRUD менеджеров и курьеров (сотрудники)
- Просмотр справочника ролей и статусов заказов
- Загрузка аватаров и фотографий
- Полный доступ к заказам (как у менеджера)

## Страницы

### Вход

- **Назначение:** аутентификация по логину и паролю.
- **Элементы:** логотип Food Rush, поля логин/пароль, кнопка «Войти», ссылка на регистрацию клиента.
- **API:** `POST /auth/employees/login`
- **Макет:** [fig-16](../пояснительная-записка/assets/fig-16-prototip-stranitsy-vhoda-v-sistemu.jpg)

### Меню

- **Назначение:** список меню ресторана, активация, редактирование состава.
- **Элементы:** таблица меню; модальное окно «Редактирование меню» (название, сезонность, список блюд).
- **Действия:** создать, изменить, удалить меню; добавить/убрать блюда.
- **API:** `GET/POST /menus`, `GET/PUT/PATCH/DELETE /menus/{id}`, `GET/POST/DELETE /menus/{menuId}/dishes/{dishId}` — [dishes-and-menus.md](../api/dishes-and-menus.md)
- **Макет:** [fig-18](../пояснительная-записка/assets/fig-18-prototip-stranitsy-upravleniya-menyu.jpg)

### Блюда

- **Назначение:** каталог блюд.
- **Элементы:** таблица (название, стоимость, число ингредиентов); модальное редактирование (название, вес, калории, цена, описание, ингредиенты, фото).
- **API:** `/dishes`, `/dishes/{id}/photos`, `/dishes/{id}/ingredients`, `POST /photos` — [dishes-and-menus.md](../api/dishes-and-menus.md)
- **Макет:** [fig-19](../пояснительная-записка/assets/fig-19-prototip-stranitsy-upravleniya-blyudami.jpg)

### Ингредиенты

- **Назначение:** справочник ингредиентов для состава блюд.
- **Элементы:** таблица ингредиентов, формы создания/редактирования.
- **API:** `/ingredients` — [dishes-and-menus.md](../api/dishes-and-menus.md)
- **Макет:** в боковом меню группы «Ресторан» (см. [fig-32](../пояснительная-записка/assets/fig-32-navigatsionnaya-model-interfeysa-administratora.jpg))

### Менеджеры

- **Назначение:** учётные записи менеджеров.
- **Элементы:** таблица (фамилия, телефон); модальное окно (ФИО, телефон, email, логин, пароль, роль).
- **API:** `/employees` (фильтр по роли MANAGER) — [employees-and-roles.md](../api/employees-and-roles.md)
- **Макет:** [fig-21](../пояснительная-записка/assets/fig-21-prototip-stranitsy-upravleniya-menedzherami.jpg)

### Курьеры

- **Назначение:** учётные записи курьеров и статус «на смене».
- **Элементы:** таблица (фамилия, телефон, статус работы); модальное редактирование анкеты.
- **API:** `/employees` (роль COURIER), `PATCH` для `is_working` — [employees-and-roles.md](../api/employees-and-roles.md)
- **Макет:** [fig-20](../пояснительная-записка/assets/fig-20-prototip-stranitsy-upravleniya-kurerami.jpg)

### Настройки

- **Назначение:** параметры учётной записи сотрудника.
- **Route:** `/staff/settings`
- **API:** профиль сотрудника — `GET/PATCH /employees/{id}` (свой `user_id` из JWT)

### Справка и о разработчиках

- **Справка:** `/staff/help` — полное руководство пользователя (все роли), оглавление с якорями, иллюстрации из макетов.
- **О разработчиках:** `/staff/about` — сведения о команде и учебном заведении.
- Доступ из бокового меню (нижний блок, рядом с «Настройки»).

### Заказы (опционально для админа)

Администратор имеет те же права на заказы, что и менеджер — см. [manager.md](manager.md#заказы).

## Навигация

Боковое меню: группы «Ресторан» (Меню, Блюда, Ингредиенты), «Сотрудники» (Менеджеры, Курьеры), «Настройки», «Справка», «О разработчиках» — [fig-32](../пояснительная-записка/assets/fig-32-navigatsionnaya-model-interfeysa-administratora.jpg).
