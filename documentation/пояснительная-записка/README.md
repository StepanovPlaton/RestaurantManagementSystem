# Пояснительная записка (Markdown)

Версия документации для работы с LLM: текст разбит по разделам, рисунки вынесены в `assets/`, подписи и каталог — в `manifest.yaml`.

**Актуальная техническая документация** (API, БД, роли для frontend): [../README.md](../README.md). Расхождения с этой запиской — [../zamechaniya-k-poyasnitelnoj-zapiske.md](../zamechaniya-k-poyasnitelnoj-zapiske.md).

## Навигация

- [index.md](./index.md) — оглавление и таблица всех рисунков
- [manifest.yaml](./manifest.yaml) — машиночитаемый каталог рисунков (подписи, ключевые слова, контекст)

## Исходники

- Оригинал Word: [../Пояснительная записка.docx](../Пояснительная%20записка.docx)
- PDF: [../Пояснительная записка.pdf](../Пояснительная%20записка.pdf)

## Перегенерация из DOCX

```bash
pip install python-docx pyyaml pillow
python documentation/scripts/docx_to_markdown.py
```

Скрипт перезаписывает все `.md` в этой папке и файлы в `assets/`.

## Соглашения

- Имена файлов рисунков: `fig-{NN}-{slug}.png` (порядок появления в документе).
- В тексте «Рисунок N» — нумерация из оригинала; `fig-id` в HTML-комментарии — стабильный идентификатор файла (`fig-01` … `fig-43`).
- Записи с `draft: true` в manifest — подписи-заглушки «Рисунок ХХХ» в исходном DOCX.
