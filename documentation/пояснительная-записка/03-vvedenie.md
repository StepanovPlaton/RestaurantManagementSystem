# Введение

Ресторанный бизнес является одной из наиболее динамично развивающихся сфер услуг. Предприятия общественного питания играют важную роль в современной экономике, предоставляя населению возможность получать готовые блюда и проводить время в комфортной обстановке. Конкуренция в данной отрасли постоянно растёт, поэтому заведения стремятся повышать качество обслуживания, расширять ассортимент и внедрять современные технологии для взаимодействия с клиентами. В условиях цифровизации особое значение приобретает использование информационных систем, позволяющих оптимизировать процессы обслуживания, приёма и обработки заказов. Такие решения помогают ресторанам ускорять работу персонала, снижать количество ошибок и повышать удовлетворённость посетителей.

История автоматизации общественного питания началась с бумажных чеков и механических кассовых аппаратов. Столкнувшись с трудностями учета и человеческим фактором при передаче заказов, индустрия перешла к использованию локальных POS-терминалов. Изначально предназначенные для внутреннего контроля продаж, эти системы быстро вышли за пределы кассовой зоны и вскоре приобрели форму облачных веб-решений, доступных каждому посетителю через браузер личного устройства.

Современный ресторан обрабатывает сотни заказов за вечер, где каждый может быть изменен по предпочтениям клиента. Такое количество делает процесс ручного принятия заказов сложным и подверженным ошибкам. С развитием интернет-технологий появились приложения с возможностью отслеживания статуса в реальном времени, что расширило возможности контроля исполнения и прозрачности процесса для пользователя.

Внедрение таких систем развивает важные бизнес-показатели: скорость обслуживания, точность передачи требований на кухню и удовлетворенность клиентов. Благодаря этим качествам приложение для удаленного заказа стало не просто инструментом автоматизации, а эффективным средством повышения конкурентоспособности заведения на рынке услуг.

Во время курсового проектирования необходимо разработать автоматизированную систему удаленного создания заказов в системе ресторана и контроля их исполнения. Система позволит пользователю формировать заказ через веб-интерфейс, передавать данные на кухню в реальном времени и отслеживать статусы приготовления блюд на всех этапах выполнения заказа.

Разработка системы будет производиться по технологии быстрой разработки приложений RAD (Rapid Application Development)*,* которая поддерживается методологией структурного проектирования и включает элементы объектно-ориентированного проектирования и анализа предметной области [1].

При проектировании системы будут использоваться методология ООАП (Object-Oriented Analysis/Design), в основу которой положена объектно-ориентированная методология представления предметной области в виде объектов, являющихся экземплярами соответствующих классов, и язык моделирования UML (Unified Modeling Language), который является стандартным инструментом для разработки «чертежей» программного обеспечения [2].
